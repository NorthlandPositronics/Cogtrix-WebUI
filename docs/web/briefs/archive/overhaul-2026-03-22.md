## Mockup 1: `markdown-full-spec.svg`

**Path**: `docs/web/mockups/markdown-full-spec.svg`
**Purpose**: Visual reference for all markdown elements as they should render inside an assistant message bubble. This is the build target for `web_coder` implementing `markdownComponents.tsx`.

**Canvas**: 640px wide, white background. Show the content as it would appear inside an assistant bubble (`rounded-2xl rounded-bl-sm border border-zinc-200 bg-white px-4 py-3 shadow-sm`, max-w ~75% of the chat area).

**Content to show (top to bottom, inside the bubble):**

1. **h1** — "Heading Level 1" — `text-xl font-semibold leading-tight text-zinc-900`, with visible `mt-4` separation from preceding block
2. **h2** — "Heading Level 2" — `text-lg font-semibold leading-tight text-zinc-900`
3. **h3** — "Heading Level 3" — `text-base font-semibold leading-tight text-zinc-900`
4. **h4** — "Heading Level 4" — `text-sm font-semibold leading-tight text-zinc-700` (noticeably smaller/muted)
5. **h5** — "Heading Level 5" — `text-sm font-medium leading-tight text-zinc-600`
6. **h6** — "Heading Level 6" — `text-sm font-medium leading-tight text-zinc-500` (most muted)
7. **Paragraph** — 2 lines of lorem text — `text-base leading-relaxed text-zinc-900`
8. **Blockquote** — "This is a blockquote." — left border: 2px solid violet-200, background violet-50 at 40% opacity, padding left 16px top/bottom 4px, italic, text-zinc-600, rounded-r-sm
9. **Unordered list** — 3 items, disc bullets, pl-5, space-y-1
10. **Ordered list** — 3 items, decimal numbers, pl-5, space-y-1
11. **Horizontal rule** — thin line, border-zinc-200
12. **Image placeholder** — show as a grey rectangle (200×100) with rounded-md and my-2 spacing
13. **CodeBlock (fenced)** — "python" label — show the full header row + pre block (see §7 spec and Mockup 2 for detail)
14. **Table** — 3 columns × 3 rows — show header row (bg-zinc-50, border-b border-zinc-200), data rows with hover state implied (row 2 shown with bg-zinc-50 hover)
15. **Inline code** — within a paragraph: "Use `inline_code` here" — `rounded bg-zinc-100 px-1 py-0.5 font-mono text-sm text-zinc-900`
16. **Link** — "Visit the documentation" — `text-violet-600 underline`

**`space-y-3` gap**: Show visible 12px (3 × 4px) gaps between each top-level block element. The gaps should be visually apparent — not collapsed.

## Mockup 2: `code-block-component.svg`

**Path**: `docs/web/mockups/code-block-component.svg`
**Purpose**: Detailed visual spec for the `CodeBlock` component — the fenced code block with language label and copy button.

**Canvas**: 560px wide. Show two states: (A) default state, (B) after copy — "copied" feedback.

**State A — Default:**
- Outer wrapper: `rounded-md border border-zinc-200 bg-zinc-50`
- Header row (`flex items-center justify-between px-3 pt-2 pb-1`):
  - Left: language label "python" — `text-xs font-medium text-zinc-400 font-mono`
  - Right: copy button — ghost icon button 24×24px, `Clipboard` icon 14px (`w-3.5 h-3.5`), `text-zinc-400 hover:text-zinc-600`
- Code area: `<pre>` with `overflow-x-auto px-3 pb-3 pt-0 font-mono text-sm leading-relaxed`
- Sample code (3–4 lines of Python):
  ```python
  def greet(name: str) -> str:
      return f"Hello, {name}!"

  print(greet("world"))
  ```

**State B — Copied:**
- Same layout, but copy button icon changes to `Check` (checkmark) — `text-green-600`
- No other visual changes
- Label next to or replacing button: "Copied!" in `text-xs text-green-600` (optional — show as tooltip or inline text)

**Hover state**: Show copy button with `text-zinc-600` (darker) on hover.

**Focus state**: Show copy button with `ring-2 ring-zinc-400 ring-offset-2` focus ring.

## Mockup 3: `blockquote-corrected.svg`

**Path**: `docs/web/mockups/blockquote-corrected.svg`
**Purpose**: Side-by-side comparison of the incorrect current implementation vs the correct v3.0 spec, to give `web_coder` a clear before/after target.

**Canvas**: 640px wide, two columns.

**Left column — "Current (incorrect)":**
- `border-l-4 border-zinc-300 pl-3 text-zinc-600 italic` — no background
- Text: "This is a quoted passage."
- Note: "4px zinc-300 border, no background"

**Right column — "Correct (v3.0)":**
- `border-l-2 border-violet-200 bg-violet-50/40 pl-4 py-1 italic text-zinc-600 rounded-r-sm`
- Text: "This is a quoted passage."
- Note: "2px violet-200 border, violet-50/40 background tint, rounded-r-sm"
- The violet-50/40 tint should be clearly visible as a very light lavender background

**Color values to use:**
- `border-violet-200` = `#ddd6fe`
- `bg-violet-50/40` = `#f5f3ff` at 40% opacity (blend with white → approximately `#faf9ff`)
- `text-zinc-600` = `#52525b`
